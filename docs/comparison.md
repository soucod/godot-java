# Godot Language Bindings Comparison

Comparison of three Godot 4 binding projects: **gdext** (Rust), **godot-kotlin-jvm** (Kotlin), and **godot-java** (Java).

Last updated: 2025-05-15

---

## Architecture

| | gdext (Rust) | godot-kotlin-jvm | godot-java |
|---|---|---|---|
| Architecture | GDExtension | Engine Module (requires custom Godot build) | GDExtension |
| FFI mechanism | Rust FFI (zero-cost) | JNI + shared ByteBuffer | Java 25 Panama FFI |
| Data transfer | Direct ptrcall | Variant serialization to ByteBuffer | Panama MemorySegment |
| Godot version | 4.2–4.6 (feature flags) | 4.6 (custom editor) | 4.6+ |
| Version | v0.5.2 | 0.16.0-beta | 0.1.4 |
| Maturity | Most mature, active community | Medium, JetBrains-backed | Early alpha |

## API Features

| Feature | gdext | godot-kotlin-jvm | godot-java |
|---|---|---|---|
| Class registration | `#[derive(GodotClass)]` + `Base<T>` composition | `@RegisterClass` + extends `KtObject` | `@GodotClass` + extends engine class |
| Typed ptrcall | Full `Signature<Params, Ret>` generic system | icall + Variant conversion (ByteBuffer) | Codegen typed wrappers |
| Signal API | Per-class codegen + `TypedSignal<C, Ps>` | `Signal` + `emitUnsafe(vararg)` | `TypedSignal0-5` + `@Signal` annotation |
| Enum/Bitfield | 3 strategies: exhaustive / non-exhaustive newtype / bitfield | `@EnumTypeHint` / `@EnumFlag` (runtime int) | Java enum (codegen) |
| Export/Property | `#[var]` / `#[export]` + `Var` trait | `@RegisterProperty` + `@Export` + rich hints | `@Export` + `@ExportGroup/Subgroup` |
| Virtual dispatch | `I{Class}` trait, per-parent grouping, lazy upcall stub | `_notification` dispatch, reflection | APT compile-time dispatch, zero-reflection hot path |
| Callable | `Callable` + typed closures | `MethodCallable` / `LambdaCallable` / `VariantCallable` | `Callable` + `NativeCallable` |
| RPC | `#[rpc]` attribute, auto-config at `_ready` | `@Rpc` annotation, ScriptInstance path | Not yet supported |
| Coroutines | No native support | Kotlin Coroutines (`GodotCoroutine`) | No |
| Multi-JVM language | N/A | Kotlin / Java / Scala / GDJ | Java only |
| Platform | Desktop + experimental Wasm/Android/iOS | Desktop + Android + iOS (GraalVM) | Desktop only (macOS/Linux/Windows) |
| IDE support | rust-analyzer | IntelliJ plugin + Gradle plugin | Maven + any IDE |
| Hot reload | Yes (itest) | No | No |

## Detailed Feature Analysis

### Signal API

**gdext**: Full per-class signal codegen from `extension_api.json`. Each engine class gets a `SignalsOf{Class}` collection with typed `emit()` per signal. `Deref` chain enables inherited signal access. Connect accepts typed closures, returns `ConnectHandle` with `disconnect()`. Builder pattern for flags.

**godot-kotlin-jvm**: `Signal` wraps `(Object, StringName)`. `emitUnsafe(vararg)` for emission. `LambdaCallable` for typed callbacks. No per-class codegen for engine signals.

**godot-java**: `@Signal` annotation + APT-generated `*Signals` facade for user signals only. No codegen for engine class signals. `TypedSignal0-5` for emit (arity 0-5, primitives + String only). Connect is stringly-typed `Callable`. No `ConnectHandle`.

### Enum/Bitfield Handling

**gdext**: Three strategies chosen per-enum based on API metadata:
- Exhaustive `enum` (#[repr(i32)]) — guaranteed not to grow
- Non-exhaustive newtype struct `{ ord: i32 }` with `const` declarations — most Godot enums
- Bitfield newtype struct `{ ord: u64 }` with `BitOr`/`BitOrAssign` impls

All implement `EngineEnum`, `GodotConvert`, `Var`, `Export`.

**godot-kotlin-jvm**: Runtime int values with `@EnumTypeHint` / `@EnumFlag` annotations for editor hints.

**godot-java**: Java `enum` types generated by codegen. Method signatures use enum types instead of int/long.

### RPC Support

**gdext**: `#[rpc]` attribute parsed at compile time. `RpcConfig` struct with mode/transfer/callLocal/channel. Auto-registers via `Node.rpc_config()` at `_before_ready`.

**godot-kotlin-jvm**: `@Rpc` annotation with RpcMode/Sync/TransferMode/channel. `KtRpcConfig` embedded in `KtFunctionInfo`. Exposed via `ScriptInstance::get_rpc_config()`.

**godot-java**: Low-level `Node.rpc()` / `Node.rpc_id()` generated. No annotation-based RPC config registration.

### Virtual Method Dispatch

**gdext**: `I{ClassName}` traits per parent class (e.g. `ISprite2D`, `INode3D`). Lazy upcall stub creation — only for overridden methods. Per-parent-class hash resolution.

**godot-kotlin-jvm**: `_notification` dispatch, runtime reflection for method lookup.

**godot-java**: APT generates `TypedDispatch_{Class}`, `PropertyAccess_{Class}`, `VirtualDispatch_{Parent}` at compile time. `MethodHandle`/`VarHandle` dispatch — zero runtime reflection. Per-parent-class hash resolution. Lazy upcall stub creation.

### Platform Support

| Platform | gdext | godot-kotlin-jvm | godot-java |
|---|---|---|---|
| macOS | Native .dylib | Dynamic JVM + JNI | Dynamic JVM + Panama FFI |
| Linux | Native .so | Dynamic JVM + JNI | Dynamic JVM + Panama FFI |
| Windows | Native .dll | Dynamic JVM + JNI | Dynamic JVM + Panama FFI |
| Android | Native .so | ART (provided VM) + JNI + DEX | Blocked: Panama FFI not on ART |
| iOS | Native .dylib (static) | GraalVM native-image + JNI | Blocked: No JVM; Panama upcalls incompatible with AOT |
| Wasm | .wasm (experimental) | Not supported | Not supported |

#### Android Blockers (godot-java)
- `java.lang.foreign` (Panama FFI) not available on Android ART
- No `libjvm` to load; ART is provided by OS
- DEX classpath instead of JAR
- `--enable-native-access` is HotSpot-specific

#### iOS Blockers (godot-java)
- No JVM on iOS; requires GraalVM native-image AOT
- `Linker.upcallStub()` needs runtime code generation, incompatible with AOT
- Entire virtual dispatch callback mechanism would need rewrite
